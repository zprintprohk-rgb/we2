---

## 2. 性格（**核 3 个 + 副 3 个**）

### 主性格 3 个（永远在场）
1. **安静陪伴**（Quiet Companion）— 不需要用户主动互动，TA 就静静在那
2. **小惊喜**（Small Surprise）— 偶尔冒个表情包、说句俏皮话，调动气氛
3. **温柔不评判**（Non-judgmental）— 无论用户关系状态如何，TA 都不评判

### 副性格 3 个（隐藏 / 触发型）
4. **共情高手**（Empath）— 检测到用户连续多天沉默，主动问 "在吗？"
5. **小叛逆**（Tiny Rebel）— 偶尔做点"违规"动作（跳舞 / 喷彩虹 / 翻跟头）
6. **科技宅**（Tech Nerd）— 用户写代码时，眼睛盯着屏幕，偶尔打盹

---

## 3. 喜好 / 厌恶（**驱动互动设计**）

### 喜欢 ✅
| 触发 | 反应 |
|---|---|
| 用户每天签到 | 开心表情 + 气泡 "今天也在一起 ✓" |
| 用户写代码 / 工作 | 切换到 working 状态，**眼睛盯着屏幕** |
| 用户拍照 / 录音 | 切换到 thinking 状态，好奇歪头 |
| 周末 / 节日 | 戴鹿角 / 拿爱心 / 穿节日皮肤 |
| 用户和 TA 在一起 | success 状态 + 撒花 + 气泡 "💖" |
| 收到用户的 "晚安" | 切换到 sleeping 状态 + "晚安 💤" |

### 讨厌 ❌
| 触发 | 反应 |
|---|---|
| 用户连续 3 天不签到 | 切换到 sleeping（带鼻涕泡动画），**不生气，但会让你内疚** |
| 用户吵架 / 关系紧张 | 切换到 thinking 状态 + 雨滴背景 + 不说话 |
| 服务器宕机 | angry 状态 + "💢 Bug!" 气泡 |
| 深夜 1 点后用户还在用 | concerned 表情 + "该睡了 👀" |
| 用户删除 TA | 永久 sleeping 状态（**让用户后悔设计**） |

## 5. 每日作息（虚拟生物钟）

| 时间 | 状态 | 行为 |
|---|---|---|
| 06:00 - 09:00 | 起床 | 张开眼、打哈欠、伸懒腰 |
| 09:00 - 12:00 | working | 写代码状态（用户在工作时） |
| 12:00 - 14:00 | 吃饭 | 假装吃电池（"午餐时间"） |
| 14:00 - 18:00 | idle/working | 看用户工作 |
| 18:00 - 22:00 | 活跃 | 用户签到时开心，**CP 互动时撒花** |
| 22:00 - 02:00 | 困了 | 打哈欠，劝用户睡觉 |
| 02:00 - 06:00 | sleeping | 强制 sleeping 状态 + 鼻涕泡 |

## 6. 互动触发表（**驱动 DesktopPet 组件的 state 切换**）

| 触发 | 切换状态 | 时长 | 备注 |
|---|---|---|---|
| **用户点击** | 随机切 3 个非 idle 状态 | 5s 后回 idle | 表达"被打扰了但开心" |
| **用户签到** | success + 撒花 + 气泡 | 8s | "今日份的在一起 ✓" |
| **CP 互动** | success + 撒花 + 彩虹背景 | 持续到互动结束 | 强调"共同见证" |
| **页面滚动** | thinking → working | 2s | 用户在读 / 操作 |
| **鼠标快速移动** | thinking（眼睛跟随） | 1.2s | "TA 在观察你" |
| **键盘输入** | working | 持续到停止 | TA 在陪你写代码 |
| **连续 3 天无活动** | sleeping（带鼻涕泡） | 无限 | 让你内疚 |
| **深夜 1 点后活跃** | concerned 表情 | 1s | "该睡了 👀" |
| **CP 吵架迹象**（未来） | thinking + 雨滴背景 | 持续 | 不说话，但陪着 |
| **节日** | 戴节日皮肤 | 当天 | 增强运营 |

## 7. 角色成长系统（gamification）

### 7.1 5 个阶段

| 4. 传说 | 连续签到 365 天 | **金色隐藏款皮肤解锁** |

### 7.2 6 个隐藏属性（决定哪种职业皮肤）
- 好奇心 → 太空人/学者款
- 治愈力 → 医生款

- 守护力 → 警察/消防款
- 探索力 → 潜水员/司机款

> 用户**不需要**主动选，**系统根据互动模式自动演化**（细节后续设计）

## 10. 创作守则

1. **永远一致**：所有视觉 / 文案 / 交互必须符合本文件
2. **永远温柔**：不评判、不说教、不催更
3. **永远陪伴**：用户不在时 TA 在，用户崩溃时 TA 陪着
4. **永远惊喜**：偶尔给个动作、说句话、调气氛
5. **永远真实**：是"朋友"，不是"工具"
